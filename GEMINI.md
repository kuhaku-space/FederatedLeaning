# 開発環境と言語・パッケージ管理のガイドライン

このプロジェクトでは、効率的で再現可能な開発環境を維持するために **mise** と **uv** を使用しています。Geminiがコード生成やコマンド提案を行う際は、以下のルールに従ってください。

## 1. ツールチェーン管理 (mise)
開発ツールのバージョン管理には `mise` (旧 rt x) を使用します。

- **設定ファイル:** `.mise.toml`
- **ランタイム:** Python, Node.js 等のインストールやバージョン切り替えは `mise` を介して行います。
- **コマンド実行:** ツールが必要なコマンドを実行する場合、必要に応じて `mise run <script>` や `mise exec -- <command>` の形式を検討してください。
- **提案:** 新しいツールが必要な場合は、`mise use <tool>@<version>` を提案してください。

## 2. Python パッケージ管理 (uv)
Python のパッケージ管理、仮想環境管理、およびビルドには `uv` を使用します。`pip` や `venv` モジュールを直接呼び出すことは避けてください。

- **パッケージの追加:** `uv add <package>`
- **開発用パッケージの追加:** `uv add --dev <package>`
- **スクリプトの実行:** `uv run <script_name>.py`
- **同期:** `uv sync`
- **依存関係:** `pyproject.toml` および `uv.lock` を正しく管理してください。

## 3. Antigravity 固有のルール
- Antigravity のフレームワーク構造に基づいたコーディングを行ってください。
- 新しいモジュールや依存関係を追加する際は、必ず `uv` を使用して依存関係の整合性を保ってください。

## 4. 禁止事項
- `pip install` コマンドの提案（代わりに `uv add` を使用）。
- `python -m venv` による仮想環境作成の提案（代わりに `uv` の自動管理を利用）。
- `asdf` や `pyenv` の操作（代わりに `mise` を使用）。
